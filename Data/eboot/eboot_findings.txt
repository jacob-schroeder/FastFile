Loader routine anchors
----------------------
0xdf258 reads a 32-bit asset/XFILE version from the active stream cursor and compares it to 0x10d (269).
0xe0d68 is a fatal stream/end/error path reached when stream input cannot be advanced.
0xe0df0 refills/advances the active stream buffer. It uses state rooted at TOC offset -6648(r2).
0xe10f8 copies requested bytes out of the active stream window and refills until the request is satisfied.
0xe11c8 handles larger streaming copies in 0x10000-byte chunks and cache-aligned windows.

Stream-state observations
-------------------------
The stream state pointer is loaded from TOC offset -6648(r2).
The refill path uses 0x40000-byte windows and advances source/destination cursor fields in the stream state.
The copy path stores caller destination at streamState+0x58 and requested byte count at streamState+0x5c.
The available byte count is read from streamState+0x50, current source pointer from streamState+0x4c, and current cursor/window pointer from streamState+0x08/0x18.

Implementation impact
---------------------
The C# reader must treat zone data as big-endian.
The XFILE header is Size, ExternalSize, then MAX_XFILE_COUNT 32-bit block sizes.
The XAssetList root follows the header in the serialized stream.
Pointer raw values must keep the engine convention:
0 = null
-1 = inline
-2 = insert
other positive values = encoded stream block plus offset pointer
